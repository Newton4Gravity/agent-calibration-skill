# Contributing

Contributions are welcome.

Useful contributions include:

- new failure modes
- clearer recovery prompts
- examples for additional platforms
- translations
- shorter mobile-friendly versions
- test cases showing agent failure and recovery
- refinements to evidence labels or tool-use rules

## Principles

Please keep changes aligned with the core purpose:

1. No fabricated output.
2. Evidence before claims.
3. Assumptions must be visible.
4. Agents should act when enough safe evidence exists.
5. High-impact actions require authorization.
6. The skill should stay portable across platforms.

## Suggested issue format

Use the failure-mode report template in `.github/ISSUE_TEMPLATE/failure-mode-report.md`.
